# План работы

Ниже последовательный план, чтобы реализовывать проект шаг за шагом. Каждый пункт — отдельный этап с понятным результатом.

## Этап 1 — Согласование требований и MVP

1. Уточнить цель приложения (какие языки, уровни, формат тренировок).
2. Описать основные сущности: пользователь, модуль, урок, упражнение, прогресс.
3. Определить типы упражнений для MVP (например: карточки, перевод, выбор).
4. Согласовать требования к AI-контенту (формат, частота генерации, проверка качества).
5. Зафиксировать Telegram Mini App ограничения и UX потоки.

Результат: краткий документ с требованиями и границами MVP.

## Этап 2 — Архитектура и структура монорепо

1. Создать каркас `backend/` и `webapp/`, определить общие папки.
2. Выбрать подход к API (REST + OpenAPI).
3. Утвердить схему данных (JSON-структуры упражнений и прогресса).
4. Зафиксировать выбор AI-провайдера и способ вызова.

Результат: структура репозитория и архитектурная схема.

## Этап 3 — Бэкенд (ядро) + тесты

1. Инициализировать Go-модуль, базовую структуру пакетов.
2. Реализовать проверку Telegram WebApp init data.
3. Сервис генерации контента (мок/интерфейс для AI).
4. CRUD для упражнений и прогресса.
5. Тесты для ключевых доменных правил и API (обязательное покрытие).

Результат: рабочий API с тестами.

## Этап 4 — AI-генерация контента

1. Определить промпты и формат ответа.
2. Реализовать слой валидации/фильтрации AI-ответов.
3. Добавить кэширование/хранилище контента.
4. Написать тесты на генерацию и валидацию (моки).

Результат: стабильный генератор контента с тестами.

## Этап 5 — Webapp (Telegram Mini App)

1. Инициализировать Vite + React + TS, базовую навигацию.
2. Экран выбора уровня и модуля.
3. Экран упражнения и интерфейс ответа.
4. Экран результата и прогресса.
5. Подключить API, обработать состояния загрузки.

Результат: рабочий UI, связанный с API.

## Этап 6 — Сквозные проверки и полировка

1. Проверить сценарии от входа до завершения упражнения.
2. Добавить e2e/интеграционные тесты при необходимости.
3. Проверить корректность отображения в Telegram.
4. Документировать запуск и переменные окружения.

Результат: стабильный MVP с документацией.
